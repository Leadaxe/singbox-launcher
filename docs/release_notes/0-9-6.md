## Highlights (EN)

Two architectural deliveries land together in this release. **SPEC 057-R-N** finishes the preset-bundles story by moving outbound preset binding into state itself — bundled outbounds now persist with `ref` + `updates[]` fields, so reorder is durable, disable cleanly tears down patches, and the runtime stops re-deriving the same data from template on every refresh. Alongside it, the long-deprecated **`independent_cache` DNS option** is removed (sing-box 1.14 moved DNS cache to per-transport keying automatically). Plus a handful of UI-side fixes that turn the Outbounds tab from a fragile mirror of the build pipeline into a direct, predictable view of state.

### Details

- **DNS tab: `independent_cache` checkbox removed (sing-box 1.14 deprecation).** Upstream sing-box documents the option as deprecated in 1.14.0 — DNS cache now always keys by transport name automatically, making the toggle a no-op (and a future config-parser error). The checkbox is gone from the DNS tab, the field is no longer written to `state.json` / `config.json`, and legacy state files containing it are silently dropped on load (`independent_cache: true` in old `dns_options` is parsed and discarded; `dns_independent_cache` in `state.vars` is cleaned up on first sync). Migration is automatic — no user action needed. ([sing-box DNS docs](https://sing-box.sagernet.org/configuration/dns/), [1.14 migration guide](https://sing-box.sagernet.org/migration/#migrate-independent-dns-cache))
- **Configurator: preset outbound binding now persists in state.** When you enable the **Russian domains & IPs** preset (or any other with `preset.outbounds[]`), bundled outbounds like `ru VPN 🇷🇺` are now stored in `state.json` with a `ref: "<preset_id>"` field and locked (🔒) in the Outbounds tab. Reordering them via ↑/↓ persists across Save. Disabling the preset removes them — both from UI and from state — immediately, no Save needed. Preset `mode=update` patches (e.g. the `!RU` filter that `russian` applies to `proxy-out` / `auto-proxy-out`) now live as an explicit `updates: [{ref, patch}]` stack on the target outbound — disabling the preset cleanly removes its patch without leaving residue in the base body.
- **Outbounds tab: "Restore missing" button (🔄 icon, left of Add).** Recovers template-defined outbounds that were accidentally deleted (e.g. `auto-proxy-out`, `vpn ①`, `direct-out`). Walks `template.parser_config.outbounds[]` and appends any missing tags. Existing entries are not touched (user customizations preserved). Tooltip explains the behavior.
- **Outbounds tab: disabled subscription cascade — 4th fix point.** When a subscription is disabled, its outbound tags (`BL:auto`, `BL:select`, etc.) no longer appear in **Rules tab outbound dropdowns**. Previously they did, allowing the user to pick a dangling reference that the build pipeline would silently drop. Now all four code paths filter disabled subscriptions identically (Outbounds tab, Rules tab dropdown, available-tags computation, build cache rebuild).
- **Rules tab: library dialog "already added" presets now show a pre-checked, disabled checkbox** (same visual treatment as required DNS servers). Previously the checkbox was empty + disabled, which was easy to misread as "available to add". Now ✓ + grey = "already in list, untoggleable here".

### Technical / Internal
- **SPEC 057-R-N — preset outbound binding via `ref` + `updates[]` in state (state = single source of truth).** Closes the last architectural gap between SPEC 053 (preset bundles) / SPEC 055 (preset.outbounds) / SPEC 056-R-N (DNS resolver pattern) and the outbound section. Before this SPEC, preset-bundled outbounds were materialized at render time on every UI refresh and rebuild via `ApplyPresetOutboundsToParserConfig` — UI used a synthetic-row + in-memory display-order map kludge; reorder didn't persist; `mode=update` patches were re-derived from template each build. Now: (1) `OutboundConfig` gets `Ref string` and `Updates []OutboundUpdate{Ref, Patch}` fields with full `omitempty` backward compat; (2) `SyncOutboundsWithActivePresets` lifecycle helper (idempotent) mutates `state.connections.outbounds[]` on Load / preset toggle / before Save / in headless rebuild paths — adopt-on-first-sync handles legacy state where preset outbounds were promoted to globals; (3) `ResolveOutbounds` pure func gives merged view (`base + apply Updates in order`) parallel to `ResolveDNS` / `ResolveRoute`; (4) `MergeOutboundUpdatesInPlace` runtime helper flattens Updates stack into body before `GenerateOutboundsFromParserConfig` (generator doesn't know about Updates); (5) `ApplyPresetOutboundsToParserConfig` + 10 tests + `PresetOutboundAddTags` + `cloneParserConfig` + `outboundsIdentical` all deleted (~350 lines); (6) UI `outbounds_configurator` reads `state.connections.outbounds[]` directly — `collectPresetOutboundRows`, `OutboundDisplayOrder`, `reorderRowDisplay` and friends removed (~170 lines); (7) `RefreshAfterPresetToggle` presenter method unifies the 4-step aftermath (DNS UI + outbounds Sync + Outbounds tab refresh + RefreshOutboundOptions) — Rules tab toggle handler is now a one-liner. Bug fixes found along the way: `parseAndPreview` no longer overwrites `model.ParserConfig` with the merged copy (separate `parserConfigForGen` for the generator); `CreateStateFromModel` syncs both Connections AND ParserConfig.Outbounds views to defeat `syncConnectionsFromLegacy` adapter overwriting Sync's work; legacy state with merged body + Updates[] stack is idempotent on emit (filters replace = same, addOutbounds union = dedup). 14 new unit tests, all 24 packages green. See `SPECS/057-R-N-OUTBOUNDS_PRESET_BINDING/IMPLEMENTATION_REPORT.md` for the full report.

- **SPEC 056 — preset.outbounds via pre-patch parser_config + DNS schema cleanup (state = refs only invariant).** Two layers landed under one SPEC because they share the same architectural class. **Layer 1 — outbounds (Phase 0–8):** replaces SPEC 055 post-merge implementation (which leaked launcher-only fields `options/filters/addOutbounds/comment/wizard` into final `config.outbounds[]` and triggered sing-box 1.12+ FATAL errors on every Rebuild). New architecture converts `template.PresetOutbound[]` (typed mirror of `configtypes.OutboundConfig` plus control fields `Mode/If/IfOr`) into `configtypes.OutboundConfig` and **pre-patches** a deep-clone of `parserCfg.ParserConfig.Outbounds[]` **before** native `GenerateOutboundsFromParserConfig` runs. Native pipeline (unchanged since v0.9.5) handles options-flatten, filters → `filterNodesForSelector`, addOutbounds union, comment-as-prefix. Zero strip/sanitize/transform functions for outbound JSON. `mode=update` on parser-generated outbounds (`proxy-out`/`auto-proxy-out`/`vpn ①/②`) works trivially. Tag collision: identical body = silent skip; otherwise first-wins + warning. Wired into Rebuild / Update / wizard Preview. Route post-pass cleans dangling outbound refs in `route.rules[]` (skip in preview). Template migration moves `!RU` filter from globals into `ru-inside`/`russian`/`ru-blocked` preset.outbounds[]; `ru VPN 🇷🇺` selector is now preset-emitted (mode=add). **Layer 2 — DNS schema cleanup (Phase 9, ex-SPEC 057):** same class of bugs surfaced in DNS pipeline (description-field leak in bundled DNS servers, dangling rule_set refs in DNS rules, template DNS library not materialized, double-emit of extras when v6 state active). Root cause: `state.dns` held `extra_servers[]` and `extra_rules[]` as **inline copies of template body** — copies went stale on template evolution. Fixed by establishing **state = thin refs only** invariant: `v6.DNSConfig.ExtraServers` and `ExtraRules` removed from schema entirely; state holds only override-refs (`template_servers[tag] = {enabled}`) and scalars. Single `stripDNSWizardOnlyFields` sanitize function applied to all DNS server emit paths. `template.dns_options.servers[]` library now materialized in live build pipeline via `ParseTemplateDNSDefaults`. User inline route rules now emit directly into `route.rules[]` (no `rule_set` wrapping) — fixes `route.rule_set[N].rules[0].protocol: unknown field` for connection-level match. 24+ new unit tests; all 24 packages green; original parser_config provably immutable; state v6 schema slimmed by ~30 lines of inline-body fields.

- **SPEC 054 fix — Xray JSON array subscriptions no longer bloat state.json.** Preview-generator was line-based (split body by `\n`, take lines containing `://`). For Xray JSON array subscriptions the whole body is one JSON string with no newlines → entire ~1 MB body was stored as a single `preview_nodes[0]` entry, blowing state.json past the recommended 256 KB threshold. Now `refreshOneSubscriptionSource` dispatches on body format: JSON array → parse via existing `ParseNodesFromXrayJSONArray`, emit one URI-like preview entry per node (`<scheme>://<server>:<port>#<tag>`, ~50-150 bytes each); base64/text-line bodies keep using the legacy fast path. `nodes_count_fetched` becomes correct (was always `1` for JSON bodies, breaking UI counters and truncation). Existing bloated state.json files self-heal on next subscription refresh — no migration needed. 6 new unit tests.

- **SPEC 053 preset bundles — shipped end-to-end.** Self-contained parametrized rule presets work top-to-bottom: `template.presets[]` with typed `vars` (outbound / dns_server / enum / text / number / bool), local-tag-scoped `rule_set` / `dns_servers`, conditional fragments via `if` / `if_or`, expansion engine, build-pipeline integration (`MergePresetsIntoRoute` / `MergePresetsIntoDNS` as additive pass over legacy `MergeRouteSection` / `MergeDNSSection` — old code untouched), state v6 schema (`presets_v1`) with auto-migration from v5 (idempotent backup as `state.json.v5.bak`), Library dialog with Presets section, Rules tab tiles, dedicated edit dialog with universal var-form rendering, **Convert to user rule** one-way conversion button, DNS tab template-server override handler syncing into `state.DNSV6.TemplateServers`. Six ready-to-use presets shipped in `bin/wizard_template.json` (`private-ips-direct`, `local-lan-domains`, `bittorrent-direct`, `block-ads`, `ru-direct-preset`, `ru-inside`) covering all preset forms. State.json format auto-switches between v5 and v6 based on presence of preset-refs — pure inline/srs users stay on v5 untouched. 91 new unit tests, all 23 packages green.

## Основное (RU)

В релизе две архитектурные поставки одновременно. **SPEC 057-R-N** закрывает историю preset-bundles переводом outbound preset binding в сам state — bundled outbound'ы теперь живут в `state.json` с полями `ref` + `updates[]`, поэтому reorder сохраняется, disable чисто откатывает patches, и runtime больше не пересоздаёт одни и те же данные из template на каждый refresh. Параллельно убран давно задеприкейтенный DNS-флаг **`independent_cache`** (sing-box 1.14 перевёл DNS-кэш на per-transport keying автоматически). Плюс несколько UI-фиксов, которые превратили вкладку Outbounds из хрупкого зеркала build pipeline в прямой и предсказуемый view state'а.

### Детали

- **DNS tab: чекбокс `independent_cache` удалён (sing-box 1.14 deprecation).** Upstream sing-box задеприкейтил опцию в 1.14.0 — DNS-кэш теперь всегда keys by transport name автоматически, чекбокс стал no-op (и будущей ошибкой парсера config'а). Чекбокс убран с вкладки DNS, поле больше не пишется в `state.json` / `config.json`, legacy state-файлы с ним silently дропаются на load (`independent_cache: true` в старом `dns_options` парсится и отбрасывается; `dns_independent_cache` в `state.vars` чистится на первом sync). Миграция автоматическая — действий от юзера не требуется. ([sing-box DNS docs](https://sing-box.sagernet.org/configuration/dns/), [1.14 migration guide](https://sing-box.sagernet.org/migration/#migrate-independent-dns-cache))
- **Configurator: preset outbound binding теперь персистится в state.** При включении preset'а **Russian domains & IPs** (или любого другого с `preset.outbounds[]`) bundled outbound'ы вроде `ru VPN 🇷🇺` теперь сохраняются в `state.json` с полем `ref: "<preset_id>"` и блокируются (🔒) во вкладке Outbounds. Reorder через ↑/↓ сохраняется при Save. Disable preset'а удаляет их и из UI, и из state мгновенно, без необходимости Save. Patches от preset `mode=update` (например `!RU` фильтр, который `russian` накладывает на `proxy-out` / `auto-proxy-out`) теперь живут как явный стек `updates: [{ref, patch}]` на target outbound — disable preset'а чисто удаляет его патч без следов в base body.
- **Outbounds tab: кнопка "Restore missing" (иконка 🔄, слева от Add).** Возвращает удалённые template-defined outbound'ы (например `auto-proxy-out`, `vpn ①`, `direct-out`). Walk'ает `template.parser_config.outbounds[]` и appends отсутствующие tag'и. Существующие entries не трогаются (юзерские правки сохраняются). Tooltip объясняет поведение.
- **Outbounds tab: каскад отключённой подписки — 4-я точка фикса.** При disabled подписке её outbound-теги (`BL:auto`, `BL:select` и т.п.) больше не появляются в **outbound dropdown'ах Rules tab**. Раньше появлялись — можно было выбрать висячий reference, который build pipeline молча drop'ал. Теперь все 4 кодовые точки фильтруют disabled подписки одинаково (Outbounds tab, Rules dropdown, available-tags computation, build cache rebuild).
- **Rules tab: library dialog "already added" preset'ы теперь показывают pre-checked + disabled чекбокс** (тот же визуальный паттерн что у required DNS-серверов). Раньше — пустой + disabled чекбокс, легко мисчитывалось как "доступно для добавления". Теперь ✓ + серый = "уже в списке, нельзя переключить тут".

### Техническое / Внутреннее
- **SPEC 057-R-N — preset outbound binding через `ref` + `updates[]` в state (state = single source of truth).** Закрывает последний архитектурный разрыв между SPEC 053 (preset bundles) / SPEC 055 (preset.outbounds) / SPEC 056-R-N (DNS resolver pattern) и outbound-секцией. До этого SPEC preset-bundled outbound'ы материализовались на render time на каждом UI refresh и rebuild через `ApplyPresetOutboundsToParserConfig` — UI использовал костыль synthetic-row + in-memory display-order карту; reorder не персистил; `mode=update` патчи re-derived из template на каждый build. Теперь: (1) `OutboundConfig` получает поля `Ref string` и `Updates []OutboundUpdate{Ref, Patch}` с полным `omitempty` backward compat; (2) lifecycle helper `SyncOutboundsWithActivePresets` (идемпотентный) мутирует `state.connections.outbounds[]` на Load / preset toggle / перед Save / в headless rebuild paths — adopt-on-first-sync обрабатывает legacy state где preset outbound'ы были промоутнуты в globals; (3) pure func `ResolveOutbounds` даёт merged view (`base + apply Updates в order`) параллельно `ResolveDNS` / `ResolveRoute`; (4) runtime helper `MergeOutboundUpdatesInPlace` флэттенит Updates стек в body перед `GenerateOutboundsFromParserConfig` (generator не знает про Updates); (5) `ApplyPresetOutboundsToParserConfig` + 10 тестов + `PresetOutboundAddTags` + `cloneParserConfig` + `outboundsIdentical` все удалены (~350 строк); (6) UI `outbounds_configurator` читает `state.connections.outbounds[]` напрямую — `collectPresetOutboundRows`, `OutboundDisplayOrder`, `reorderRowDisplay` и спутники удалены (~170 строк); (7) presenter-метод `RefreshAfterPresetToggle` унифицирует 4-шаговый aftermath (DNS UI + outbounds Sync + Outbounds tab refresh + RefreshOutboundOptions) — Rules tab toggle handler теперь однострочник. Bug'и, найденные по дороге: `parseAndPreview` больше не перезаписывает `model.ParserConfig` мерженной копией (отдельный `parserConfigForGen` для generator'а); `CreateStateFromModel` Sync'ает обе view'а Connections И ParserConfig.Outbounds для защиты от `syncConnectionsFromLegacy` адаптера; legacy state с merged body + Updates[] стеком идемпотентен на эмите (filters replace = same, addOutbounds union = dedup). 14 новых unit-тестов, все 24 пакета зелёные. См. `SPECS/057-R-N-OUTBOUNDS_PRESET_BINDING/IMPLEMENTATION_REPORT.md` для полного отчёта.

- **SPEC 056 — preset.outbounds через pre-patch parser_config + DNS schema cleanup (state = refs only invariant).** Два слоя под одним SPEC — общий архитектурный класс. **Слой 1 — outbounds (Phase 0–8):** замена SPEC 055 post-merge реализации (утекала launcher-only поля `options/filters/addOutbounds/comment/wizard` в финальный `config.outbounds[]`, валила sing-box 1.12+ FATAL на каждом Rebuild). Новая архитектура: `template.PresetOutbound[]` (типизированное зеркало `configtypes.OutboundConfig` + контрол-поля `Mode/If/IfOr`) конвертится в `configtypes.OutboundConfig` и **pre-patch'ится** в **deep-clone** `parserCfg.ParserConfig.Outbounds[]` **до** запуска native `GenerateOutboundsFromParserConfig`. Нативный pipeline (без изменений с v0.9.5) сам делает options-flatten, filters → `filterNodesForSelector`, addOutbounds union, comment-as-prefix. Ноль strip/sanitize/transform функций для outbound JSON. `mode=update` на parser-generated outbound'ах (`proxy-out`/`auto-proxy-out`/`vpn ①/②`) работает тривиально. Collision: identical body = silent skip, иначе first-wins + warning. Подключено в Rebuild / Update / визард Preview. Route post-pass чистит dangling outbound refs (skip в preview). Template-миграция: `!RU` фильтр в `ru-inside`/`russian`/`ru-blocked` preset.outbounds[]; `ru VPN 🇷🇺` теперь preset-emitted. **Слой 2 — DNS schema cleanup (Phase 9, ex-SPEC 057):** тот же класс багов в DNS pipeline (description leak в bundled DNS servers, dangling rule_set refs в DNS rules, template DNS library не материализована, double-emit extras при v6 state). Корневая причина: `state.dns` содержал `extra_servers[]` и `extra_rules[]` как **inline-копии тел template-сущностей** — копии устаревали при template-эволюции. Исправлено через инвариант **state = thin refs only**: поля `v6.DNSConfig.ExtraServers` и `ExtraRules` удалены из схемы полностью; state хранит только override-refs (`template_servers[tag] = {enabled}`) и scalars. Единая функция `stripDNSWizardOnlyFields` применяется ко всем путям DNS server emit. Библиотека `template.dns_options.servers[]` теперь материализуется в live build pipeline через `ParseTemplateDNSDefaults`. User inline route rules эмитятся напрямую в `route.rules[]` (без `rule_set` обёртки) — фикс `route.rule_set[N].rules[0].protocol: unknown field` для connection-level match. 24+ новых unit-тестов; все 24 пакета зелёные; оригинальный parser_config доказуемо immutable; v6-схема state'а похудела на ~30 строк inline-body полей.

- **SPEC 054 fix — Xray JSON подписки больше не раздувают state.json.** Preview-генератор был line-based (split body по `\n`, выборка строк с `://`). Для Xray JSON array body — это одна JSON-строка без переводов строк → весь ~1 MB body попадал в `preview_nodes[0]` как одно значение, раздувая state.json за рекомендованный порог в 256 KB. Теперь `refreshOneSubscriptionSource` диспатчится по формату body: JSON array → парсим через уже-существующий `ParseNodesFromXrayJSONArray`, эмитим по одному URI-like entry на ноду (`<scheme>://<server>:<port>#<tag>`, ~50-150 байт каждый); base64/text-line идут по legacy быстрому пути. `nodes_count_fetched` стал корректным (раньше для JSON body всегда `1`, что ломало UI-счётчики и truncation). Уже раздутые state.json самоочищаются на следующем refresh подписки — миграция не нужна. 6 новых unit-тестов.

- **SPEC 053 preset bundles — выкачен полностью.** Self-contained параметризованные пресеты правил работают end-to-end: `template.presets[]` с типизированными `vars` (outbound / dns_server / enum / text / number / bool), локально-scoped `rule_set` / `dns_servers`, условные фрагменты через `if` / `if_or`, expansion engine, **интеграция в основной build pipeline** через `MergePresetsIntoRoute` / `MergePresetsIntoDNS` (дополнительный pass поверх legacy `MergeRouteSection` / `MergeDNSSection` — старый код не трогается), state v6 схема (`presets_v1`) с авто-миграцией с v5 (идемпотентный backup `state.json.v5.bak`), UI Library dialog с секцией Presets, tile'ы preset-ref'ов в Rules tab, dedicated edit dialog с универсальным rendering формы, **кнопка Convert to user rule** для one-way конверсии, DNS tab template-server override handler синхронизируется в `state.DNSV6.TemplateServers`. В `bin/wizard_template.json` добавлено 6 ready-to-use preset'ов (`private-ips-direct`, `local-lan-domains`, `bittorrent-direct`, `block-ads`, `ru-direct-preset`, `ru-inside`) покрывающих все формы. Формат state.json авто-переключается между v5 и v6 по наличию preset-ref'ов — юзеры с чистыми inline/srs правилами остаются на v5 без изменений. 91 новый unit-тест, все 23 пакета зелёные.
