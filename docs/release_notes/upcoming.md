### Technical / Internal
- **SPEC 056 — preset.outbounds via pre-patch parser_config.** Replaces the post-merge implementation of SPEC 055 (which leaked launcher-only fields `options/filters/addOutbounds/comment/wizard` into the final `config.outbounds[]` and triggered sing-box 1.12+ FATAL errors on every Rebuild). New architecture: `template.PresetOutbound[]` (typed mirror of `configtypes.OutboundConfig` plus control fields `Mode/If/IfOr`) is converted into `configtypes.OutboundConfig` and pre-patched onto a deep-clone of `parserCfg.ParserConfig.Outbounds[]` **before** the native `GenerateOutboundsFromParserConfig` runs. Native pipeline (unchanged since v0.9.5) handles options-flatten, filters → `filterNodesForSelector`, addOutbounds union, comment-as-prefix. Zero strip/sanitize/transform functions for outbound JSON. `mode=update` on parser-generated outbounds (`proxy-out`, `auto-proxy-out`, `vpn ①/②`) works trivially. Tag collision: identical body = silent skip, otherwise first-wins + warning (template global vs earlier preset). `mode=update` on missing tag = warning + no-op (no auto-create). Wired into all three pipeline entry points (Rebuild / Update / wizard Preview). Route post-pass cleans dangling outbound refs in `route.rules[]` against the patched outbound set (skip in preview). Template migration moves `!RU` filter from globals into `ru-inside`/`russian`/`ru-blocked` preset.outbounds[], and `ru VPN 🇷🇺` selector is now preset-emitted (mode=add). 24+ new unit tests; all 24 packages green; original parser_config is provably immutable across rebuild (disable preset → effect fully disappears).

- **SPEC 054 fix — Xray JSON array subscriptions no longer bloat state.json.** Preview-generator was line-based (split body by `\n`, take lines containing `://`). For Xray JSON array subscriptions the whole body is one JSON string with no newlines → entire ~1 MB body was stored as a single `preview_nodes[0]` entry, blowing state.json past the recommended 256 KB threshold. Now `refreshOneSubscriptionSource` dispatches on body format: JSON array → parse via existing `ParseNodesFromXrayJSONArray`, emit one URI-like preview entry per node (`<scheme>://<server>:<port>#<tag>`, ~50-150 bytes each); base64/text-line bodies keep using the legacy fast path. `nodes_count_fetched` becomes correct (was always `1` for JSON bodies, breaking UI counters and truncation). Existing bloated state.json files self-heal on next subscription refresh — no migration needed. 6 new unit tests.

- **SPEC 053 preset bundles — shipped end-to-end.** Self-contained parametrized rule presets work top-to-bottom: `template.presets[]` with typed `vars` (outbound / dns_server / enum / text / number / bool), local-tag-scoped `rule_set` / `dns_servers`, conditional fragments via `if` / `if_or`, expansion engine, build-pipeline integration (`MergePresetsIntoRoute` / `MergePresetsIntoDNS` as additive pass over legacy `MergeRouteSection` / `MergeDNSSection` — old code untouched), state v6 schema (`presets_v1`) with auto-migration from v5 (idempotent backup as `state.json.v5.bak`), Library dialog with Presets section, Rules tab tiles, dedicated edit dialog with universal var-form rendering, **Convert to user rule** one-way conversion button, DNS tab template-server override handler syncing into `state.DNSV6.TemplateServers`. Six ready-to-use presets shipped in `bin/wizard_template.json` (`private-ips-direct`, `local-lan-domains`, `bittorrent-direct`, `block-ads`, `ru-direct-preset`, `ru-inside`) covering all preset forms. State.json format auto-switches between v5 and v6 based on presence of preset-refs — pure inline/srs users stay on v5 untouched. 91 new unit tests, all 23 packages green.

### Техническое / Внутреннее
- **SPEC 056 — preset.outbounds через pre-patch parser_config.** Замена post-merge реализации SPEC 055 (которая утекала launcher-only поля `options/filters/addOutbounds/comment/wizard` в финальный `config.outbounds[]` и валила sing-box 1.12+ FATAL на каждом Rebuild). Новая архитектура: `template.PresetOutbound[]` (типизированное зеркало `configtypes.OutboundConfig` + контрол-поля `Mode/If/IfOr`) конвертится в `configtypes.OutboundConfig` и pre-patch'ится в **deep-clone** `parserCfg.ParserConfig.Outbounds[]` **до** запуска native `GenerateOutboundsFromParserConfig`. Нативный pipeline (без изменений с v0.9.5) сам делает options-flatten, filters → `filterNodesForSelector`, addOutbounds union, comment-as-prefix. Ноль strip/sanitize/transform функций для outbound JSON. `mode=update` на parser-generated outbound'ах (`proxy-out`, `auto-proxy-out`, `vpn ①/②`) работает тривиально. Tag-collision: identical body = silent skip, иначе first-wins + warning (template global vs earlier preset). `mode=update` на missing tag = warning + no-op (no auto-create). - **SPEC 054 fix — Xray JSON подписки больше не раздувают state.json.** Preview-генератор был line-based (split body по `\n`, выборка строк с `://`). Для Xray JSON array body — это одна JSON-строка без переводов строк → весь ~1 MB body попадал в `preview_nodes[0]` как одно значение, раздувая state.json за рекомендованный порог в 256 KB. Теперь `refreshOneSubscriptionSource` диспатчится по формату body: JSON array → парсим через уже-существующий `ParseNodesFromXrayJSONArray`, эмитим по одному URI-like entry на ноду (`<scheme>://<server>:<port>#<tag>`, ~50-150 байт каждый); base64/text-line идут по legacy быстрому пути. `nodes_count_fetched` стал корректным (раньше для JSON body всегда `1`, что ломало UI-счётчики и truncation). Уже раздутые state.json самоочищаются на следующем refresh подписки — миграция не нужна. 6 новых unit-тестов.

- **SPEC 053 preset bundles — выкачен полностью.** Self-contained параметризованные пресеты правил работают end-to-end: `template.presets[]` с типизированными `vars` (outbound / dns_server / enum / text / number / bool), локально-scoped `rule_set` / `dns_servers`, условные фрагменты через `if` / `if_or`, expansion engine, **интеграция в основной build pipeline** через `MergePresetsIntoRoute` / `MergePresetsIntoDNS` (дополнительный pass поверх legacy `MergeRouteSection` / `MergeDNSSection` — старый код не трогается), state v6 схема (`presets_v1`) с авто-миграцией с v5 (идемпотентный backup `state.json.v5.bak`), UI Library dialog с секцией Presets, tile'ы preset-ref'ов в Rules tab, dedicated edit dialog с универсальным rendering формы, **кнопка Convert to user rule** для one-way конверсии, DNS tab template-server override handler синхронизируется в `state.DNSV6.TemplateServers`. В `bin/wizard_template.json` добавлено 6 ready-to-use preset'ов (`private-ips-direct`, `local-lan-domains`, `bittorrent-direct`, `block-ads`, `ru-direct-preset`, `ru-inside`) покрывающих все формы. Формат state.json авто-переключается между v5 и v6 по наличию preset-ref'ов — юзеры с чистыми inline/srs правилами остаются на v5 без изменений. 91 новый unit-тест, все 23 пакета зелёные.
