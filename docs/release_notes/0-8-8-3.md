# What's New from 0.8.8.1

## Version 0.8.8.3 Release Notes — Hotfix follow-up

Cleanup of the v0.8.8.1 hotfix. **v0.8.8.1 already fixed the critical bug** (sing-box couldn't start on fresh installs because `@urltest_*` literals leaked into `config.json`). This release fixes a *degraded feature* — Settings-tab user overrides for URLTest were silently dropped because the override loader had three bugs in a row (wrong path, wrong shape, wrong JSON key). The launcher worked fine through the template-default fallback for the ~99% of users who don't customize URLTest values; only the ~1% who do were affected. This release makes the override path actually work, plus a broader path-invariant refactor so this kind of thing can't happen again.

> Note: version v0.8.8.2 was used internally during work on this hotfix and never reached a published release — the third override-loader bug (wrong JSON key) was surfaced on review before the tag was created. 0.8.8.3 is the first follow-up published after v0.8.8.1.

## EN

### Fixed (degraded feature, not a critical bug)

- **`@varname` Settings-tab override now actually applies.** v0.8.8.1's `BuildVarSubstituterFromDisk` had three independent bugs in the override branch (`loadStateSettingsVars`) that together silently dropped user overrides — `urltest_interval = 1m` set in the UI never made it into the generated config. The other two resolution levels (template `default_value`, hard-coded fallback) were intact, so sing-box still got valid values; users who didn't customize URLTest saw no breakage. But anyone who did customize quietly got the template default instead of their chosen value. The three bugs:
  1. **Wrong file path.** v0.8.8.1 looked at `bin/state.json`; actual path is `bin/wizard_states/state.json`.
  2. **Wrong JSON shape.** The list was decoded as `map[string]string`; on disk it's `[{name, value}, ...]`.
  3. **Wrong JSON key.** The block was looked up under `"settings_vars"` / `"SettingsVars"`; the wizard writes it under **`"vars"`** (see `WizardStateFile.Vars` with `json:"vars"` in `ui/wizard/models/wizard_state_file.go`).

  v0.8.8.3 fixes all three. The hard-coded URLTest fallback table in v0.8.8.1 covered the visible breakage so nobody hit a regression versus v0.8.8 (which was completely broken), but anyone who customized URLTest values just got the template default.

  A new test (`varsubst_test.go::TestBuildVarSubstituterFromDisk_RealFixture`) loads `core/config/testdata/state-real.json` — captured from an actual install — and asserts overrides apply. If somebody refactors the loader and breaks the JSON tag again, this test fails loudly. No legacy key aliases (`settings_vars` / `SettingsVars`): any other top-level key would be a refactor mistake upstream and should fail loudly, not be silently absorbed.

### Changed — Path invariants

To prevent the above (a casual reader had to grep three packages to find where `state.json` actually lives), the canonical paths are consolidated:

- `internal/constants/constants.go` gains `WizardTemplateFileName`, `WizardStateFileName`, `WizardStatesDirName`. Previously these lived in `ui/wizard/template/loader.go`, `ui/wizard/models/wizard_state_file.go`, and `ui/wizard/business/state_store.go` respectively.
- `internal/platform/platform_common.go` gains three helpers — `GetWizardTemplatePath(execDir)`, `GetWizardStatesDir(execDir)`, `GetWizardStatePath(execDir)`. Code that needs to locate these files MUST go through them; no more hand-rolling `filepath.Join(execDir, "bin", "wizard_states", "state.json")` per call site.
- Existing constants in the `ui/wizard/...` packages are kept as re-exports of the canonical values (`StateFileName = constants.WizardStateFileName`, etc.) so external callers don't break.
- All existing call sites migrated to the helpers (`ui/wizard/template/loader.go`, `ui/wizard/business/state_store.go`, `core/config/varsubst.go`).

Tests pin the helpers (`platform_common_test.go`) and check the wizard state file via the real on-disk schema (`varsubst_test.go::TestBuildVarSubstituterFromDisk_StateOverrideRealSchema`).

### Technical / Internal

- `core/config/varsubst.go::BuildVarSubstituterFromDisk` signature changed from `(binDir string)` to `(execDir string)`. Internal — only `core/config_service.go` calls it.
- New `settingVarEntry` struct in `varsubst.go` mirrors `ui/wizard/models.SettingVar` JSON layout (kept local to avoid an upward import from `core/config/` to the ui-side package).

## RU

### Исправлено (degraded feature, не критический баг)

> v0.8.8.1 уже устранил критический баг (sing-box не стартовал на свежей установке из-за литералов `@urltest_*` в config.json). Этот релиз чинит **второстепенную проблему**: Settings tab override молча терялся.

- **Override `@varname` из Settings tab теперь действительно применяется.** В v0.8.8.1 `BuildVarSubstituterFromDisk` имел три независимых бага в override-ветке (`loadStateSettingsVars`), и кастомизация из UI молча игнорировалась — `urltest_interval = 1m` из Settings tab не доезжал до сгенерированного конфига. Остальные два уровня разрешения (template `default_value`, hardcoded fallback) работали корректно, поэтому sing-box получал валидные значения; ~99% юзеров (которые не трогают URLTest defaults) не пострадали. Но любой, кто кастомизировал, тихо получал template default вместо своего значения. Три бага:
  1. **Неправильный путь к файлу.** v0.8.8.1 искал `bin/state.json`; реальный путь — `bin/wizard_states/state.json`.
  2. **Неправильный формат JSON.** Список декодировался как `map[string]string`; на диске — массив `[{name, value}, ...]`.
  3. **Неправильный JSON-ключ.** Блок искался под `"settings_vars"` / `"SettingsVars"`; wizard пишет под **`"vars"`** (см. `WizardStateFile.Vars` с тегом `json:"vars"` в `ui/wizard/models/wizard_state_file.go`).

  v0.8.8.3 чинит все три. Новый тест (`varsubst_test.go::TestBuildVarSubstituterFromDisk_RealFixture`) загружает `core/config/testdata/state-real.json` — снимок с реальной установки — и проверяет, что override применяется. Если кто-то снова промахнётся JSON-тегом, тест упадёт громко. Без legacy-aliases (`settings_vars` / `SettingsVars`): любой другой ключ — это refactor-ошибка, и она должна падать громко, а не молча.

### Изменено — Инварианты путей

Чтобы такое не повторялось (читателю приходилось grep'ать три пакета, чтобы понять, где реально лежит `state.json`), канонические пути сведены:

- `internal/constants/constants.go` получил `WizardTemplateFileName`, `WizardStateFileName`, `WizardStatesDirName`. Раньше они лежали в `ui/wizard/template/loader.go`, `ui/wizard/models/wizard_state_file.go` и `ui/wizard/business/state_store.go` соответственно.
- `internal/platform/platform_common.go` получил три helper-а — `GetWizardTemplatePath(execDir)`, `GetWizardStatesDir(execDir)`, `GetWizardStatePath(execDir)`. Код, которому нужны эти файлы, ОБЯЗАН ходить через них; никаких больше самосборок `filepath.Join(execDir, "bin", "wizard_states", "state.json")` на каждом callsite.
- Существующие константы в `ui/wizard/...` пакетах сохранены как re-exports канонических значений (`StateFileName = constants.WizardStateFileName` и т.д.), чтобы внешние callers не сломались.
- Все существующие callsite-ы мигрированы на хелперы (`ui/wizard/template/loader.go`, `ui/wizard/business/state_store.go`, `core/config/varsubst.go`).

Тесты пинят хелперы (`platform_common_test.go`) и проверяют чтение wizard state по реальной on-disk схеме (`varsubst_test.go::TestBuildVarSubstituterFromDisk_StateOverrideRealSchema`).

### Техническое / Внутреннее

- Сигнатура `core/config/varsubst.go::BuildVarSubstituterFromDisk` изменена с `(binDir string)` на `(execDir string)`. Внутренняя — вызывает только `core/config_service.go`.
- Новая структура `settingVarEntry` в `varsubst.go` зеркалит `ui/wizard/models.SettingVar` JSON-layout (локально, чтобы не тянуть импорт из core/config/ в ui-слой).
