# Issues — баги и мелочи (Spec Kit)

Issue — это **папка** `todo/issues/NNN-short-name/` с тем же набором документов, что и у фичи: **SPEC → PLAN → TASKS → IMPLEMENTATION_REPORT**. Изначально это заявка на баг или мелкое улучшение; при принятии в работу заполняются PLAN и TASKS, после реализации — IMPLEMENTATION_REPORT.

## Формат папки

| Файл | Назначение |
|------|------------|
| **SPEC.md** | Проблема (симптомы, воспроизведение, гипотезы), цель исправления, требования, критерии приёмки. Создаётся при фиксации бага. |
| **PLAN.md** | Как исправлять — компоненты, изменения в файлах. Заполняется при принятии в работу. |
| **TASKS.md** | Чеклист задач по этапам. Заполняется при принятии в работу. |
| **IMPLEMENTATION_REPORT.md** | Отчёт после реализации — статус, изменения, дата. Заполняется по завершении. |

Нумерация сквозная: 001, 002, … Имя папки: `NNN-kebab-case-name`.

## Workflow

1. Завести папку `todo/issues/NNN-short-name/`.
2. Написать **SPEC.md** (симптомы, воспроизведение, гипотезы, требования к исправлению, критерии приёмки).
3. Когда берёте в работу — добавить **PLAN.md** и **TASKS.md**.
4. Реализовать по TASKS с учётом [../IMPLEMENTATION_PROMPT.md](../IMPLEMENTATION_PROMPT.md), заполнить **IMPLEMENTATION_REPORT.md**.

Папка может оставаться только с SPEC.md, пока issue не принят в работу.
